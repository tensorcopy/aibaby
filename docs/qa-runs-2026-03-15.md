# QA Runs - March 15, 2026

This note records four hands-on QA passes against the local mobile app and local development API.

## Run 1: App launch + baby profile

Scope:
- launch the mobile app in the iOS simulator
- open the baby profile route
- verify that profile reads survive a write in local dev

What I tested:
- Expo launch into the chat-first home
- deep link into `/baby-profile`
- `POST /api/babies`
- `GET /api/babies`
- `GET /api/babies/:babyId`

Issues found:
- Bug: local dev profile writes were being stored under the compiled Next.js server output, so `POST /api/babies` succeeded but follow-up reads returned `Baby profile not found`
- Bug: a stale explicit `babyId` sent the profile route into a hard error state instead of recovering to the owner-scoped current profile or create flow

Fix status:
- fixed
